## 本轮已落地到真实数据层

- 已新增真实数据库模型：
  - `CoinAccount`
  - `CoinLedger`
  - `CoinPackage`
  - `CoinRechargeOrder`
- 后台 `finance` 页已接入真实数据：
  - 真实球币套餐读取
  - 真实充值订单读取
  - 手动建单
  - 补单入账、失败、关闭、退款
  - 手动加币 / 扣币
  - 球币账户余额榜
  - 球币流水明细视图
  - 充值对账总览
  - 对账问题中心（问题创建、处理中、已解决、忽略、重开）
  - 对账异常扫描（超时待支付 / 支付失败 / 已退款 / 已支付未入账）
  - 对账 SLA 展示与负责人分配
  - 过期充值单清理入口
- 充值订单补单后会真实给用户账户入账，并生成球币流水
- 充值订单退款会尝试回退球币余额，并生成退款流水
- 后台人工调账会直接更新 `CoinAccount`，并写入 `CoinLedger`
- 已新增内部定时接口 `/api/internal/finance/recharge-expiry`，可由服务器 cron 自动关闭超时未处理充值单
- 已新增内部定时接口 `/api/internal/finance/reconciliation-scan`，可由服务器 cron 自动扫描充值对账异常
- 充值单在补单成功 / 标记失败 / 关闭 / 退款后，会自动同步并收口对应的对账问题状态
- 前台已接入球币消费链路：
  - `/plans` 列表与详情已切换为球币优先购买
  - `/plans/[slug]` 详情支持球币解锁与结果提示
  - `/member` 会员中心支持球币余额、最近流水、最近充值订单展示
  - 球币解锁内容后会复用正式 `ContentOrder` 权限链路，已购状态即时生效
- 已新增前台聚合内容接口：
  - `/api/site/banners`
  - `/api/site/announcements`
  - `/api/site/content-data`
- 已补后台单用户工作台里的账号资料字段：
  - 邮箱验证状态
  - 待更新邮箱
  - 联系方式
  - 偏好语言
  - 国家 / 地区

## 仍需继续深化的点

- 球币系统已补到真实 `球币账户 + 流水 + 充值订单 + 前台内容消费 + 批量运营加币/扣币 + 异常充值单批处理 + 财务对账导出 + 批处理结果审计 + 报表异步导出任务中心 + 对账问题中心`，但还可继续深化 `自动规则发现、更多订单类型覆盖、对账 SLA 与催办`
- 代理招商模块已补到真实数据层与后台可操作界面，代理业绩聚合看板已补到后台 `agents / reports` 双入口，并已补内部打款回调占位接口 `/api/internal/agents/payout-callback` 与后台回调字段展示；但仍未接入真实自动打款供应商
- 系统安全模块已补到真实数据层：
  - `AdminRolePolicy`
  - `AdminAuditLog`
  - `SystemAlertChannel`
  - `SystemAlertEvent`
  - `SystemParameter`
- 后台 `system` 页已接入真实数据：
  - DB 驱动 RBAC 策略配置
  - 审计日志落库与展示
  - 告警通道 / 告警事件录入与关闭
  - 系统参数维护
- 代理模块已补一期闭环：
  - 邀请链接已可落到 `/register?invite=CODE`
  - 登录 / 注册会绑定代理邀请码归因
  - 充值订单支付后会自动生成代理佣金流水
  - 充值退款后会自动冲回代理佣金
  - 后台代理页可查看归因佣金流水
- 代理模块已补二期结算闭环：
  - 提现结算会按 FIFO 占用可结算佣金，并生成 `AgentWithdrawalAllocation` 分配记录
  - 后台可维护打款账户、打款渠道、流水号、操作人与凭证 URL
  - 提现审核队列可直接查看结算分配明细
  - 内部回调可回写 `payoutBatchNo / callbackStatus / callbackPayload / callbackReceivedAt`
- 代理模块已补三级链路里的下级分佣：
  - 同一充值单可同时生成 `直推佣金 + 上级下级分佣`
  - 充值退款时会同步冲回直推与下级分佣流水
  - 后台佣金流水与导出已可区分 `direct / downstream`
- 报表中心已补代理专项导出：
  - `agent-commissions` 导出代理佣金流水、已结算/已冲回/可结算金额
  - `agent-performance` 导出代理层级、归因用户、累计佣金与提现表现
- 报表中心当前已补核心代理 BI 聚合指标、排行榜展示与异步导出任务中心，但尚未接 `更长周期聚合任务、分时趋势图`
- 会员详情页仍需继续做单用户深挖页，补齐认证资料、球币流水、已购文章、代理关系、登录日志的真实详情
- 赛事补录和联赛管理目前为运营控制台视图，后续应补正式 CRUD 和持久化接口

## 后续优先级建议

1. 先补更重的财务对账中心
2. 再把代理体系深化到 `自动打款通道接入、提现导出、回单异常监控`
3. 再做会员单用户深挖页和代理归因详情联动
4. 最后把报表中心升级成真实聚合和分时趋势图
